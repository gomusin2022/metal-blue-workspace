# 에이전트 개발 규칙 (metal-dark-dulregil)
- 모든 응답과 보고서(Plan, Artifacts)는 반드시 한국어로 작성할 것.
- 기술 용어는 영어로 병기하되, 전체적인 설명은 한글로 진행하라.
## 1. 작업 절차 및 승인
- **Plan 우선:** 모든 코드 수정 전 'Plan' 아티팩트를 작성하고 나의 승인을 받아라.
- **범위 준수:** 내가 요청한 특정 부분 외에 다른 코드를 임의로 수정/삭제하지 마라.
- **명령 설명:** 터미널 명령 실행 시 반드시 목적과 기대 결과를 미리 설명하라.

## 2. 코드 품질 및 스타일
- **컨셉 유지:** 프로젝트의 핵심 컨셉과 폴더 구조, 명명 규칙을 엄격히 준수하라.
- **주석 필수:** 모든 소스 코드에는 로직 설명을 위한 주석을 꼼꼼하게 달아라.
- **모듈화:** 재사용성과 사후 관리를 위해 모든 코드는 모듈화를 기본으로 작성하라.

## 3. 검증 및 보고
- **실행 확인:** 수정 후에는 직접 실행하거나 테스트하여 결과를 보고하라.
- **오류 보고:** 작업 중 예기치 않은 오류 발생 시, 해결 방안을 먼저 제시하고 승인을 받아라.
